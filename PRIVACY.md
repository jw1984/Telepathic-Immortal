# Privacy Policy

**Project:** Telepathic-Immortal
**Project author:** Jesse Wyer
**Effective date:** July 13, 2026
**Last updated:** July 13, 2026

## 1. Overview

Telepathic-Immortal is an experimental, general-audience art and game project. Some pages use optional microphone input to control visual effects, game movement, or other audio-reactive features.

This policy describes the intended operation of the project’s source code. It does not control independent data processing performed by a web-hosting provider, internet service provider, operating system, browser, browser extension, or other third party.

The project’s application code is designed to minimize the collection, storage, and transmission of personal information.

## 2. Microphone access

Microphone access is optional and is off by default.

A microphone-enabled page should request access only after the user:

1. Presses a clearly labeled **Start Microphone** button.
2. Responds to the browser’s microphone permission request.
3. Completes any consent acknowledgment displayed by the page.

Clicking the page background, game canvas, keyboard controls, decorative graphics, or unrelated controls should not activate the microphone.

The project also provides, or is intended to provide, a silent demonstration mode when practical so that users can view or play the project without granting microphone access.

## 3. How microphone input is used

In the current published design, microphone-enabled pages use temporary live audio values for purposes such as:

* Measuring general frequency activity.
* Measuring volume or signal intensity.
* Moving a fictional game character.
* Activating visual effects.
* Changing fictional game or animation states.

The application code is designed to process these values temporarily within the user’s browser.

The application is not designed to:

* Create an audio recording.
* Save microphone audio to a file.
* Upload microphone audio.
* Transmit microphone audio to the project author.
* Store a history of microphone activity.
* Convert speech into text.
* Determine what a person is saying.
* Identify a speaker.
* Authenticate a person by voice.
* Create or store a voiceprint.
* Recognize a returning user from the user’s voice.

Temporary audio values may remain in the browser’s active memory while a microphone-enabled feature is running. They are intended to be discarded when microphone processing stops, the page is closed, or the page is reloaded.

## 4. Stopping microphone access

Users can stop microphone processing by pressing the page’s **Stop Microphone** button.

The application is also designed to stop its active microphone tracks when the user closes the page, leaves the page, or places the page in a hidden state.

Users can independently revoke microphone permission through their browser or operating-system privacy settings.

The browser may remember a previous permission choice. Browser permission records are controlled by the browser and are not controlled by the project author.

## 5. Private conversations

Do not activate the microphone where another person’s private conversation could be captured unless everyone involved has agreed.

The microphone feature is intended for voluntary use by a person who understands that nearby sounds may enter the device’s microphone.

The application is not intended for:

* Secretly monitoring another person.
* Intercepting private conversations.
* Recording conversations.
* Surveillance.
* Workplace monitoring.
* Law-enforcement use.
* Evidence collection.
* Monitoring tenants, employees, patients, students, or customers.

Users are responsible for following the privacy and recording laws that apply where they use the project.

## 6. Information the project does not intentionally request

The current project is not designed to ask users to provide:

* A legal name.
* A username or password.
* An email address through the application.
* A telephone number.
* A mailing address.
* An age or date of birth.
* A government identification number.
* Financial or payment information.
* A photograph or video.
* An uploaded audio recording.
* Health records.
* Location information.
* Contact lists.
* Social-media account information.

The current project does not provide user accounts or user profiles.

## 7. Analytics, advertising, cookies, and tracking

As of the effective date of this policy, the project’s application code is not intentionally designed to include:

* Advertising.
* Behavioral advertising.
* Advertising profiles.
* Analytics services.
* Tracking pixels.
* Cross-site tracking.
* Marketing cookies.
* User fingerprinting.
* Persistent tracking identifiers.
* Data brokers.

This statement applies to the project code published by the project author. A website host, browser, operating system, browser extension, network provider, or other third party may independently use logs, cookies, security systems, or technical identifiers under its own terms and privacy policies.

When a user visits a hosted copy of the project, the hosting provider may process ordinary technical information needed to deliver and protect the website. Depending on the provider, this may include:

* Internet Protocol address.
* Browser type.
* Device or operating-system information.
* Request date and time.
* Requested page or file.
* Referring page.
* Security and diagnostic information.

The project author may not receive or control all information processed independently by the hosting provider.

When the project is hosted through GitHub or GitHub Pages, GitHub’s own terms and privacy documentation apply to GitHub’s processing activities.

## 9. No speaker identification or biometric system

Telepathic-Immortal is not designed as a biometric identification system.

The project does not intentionally:

* Create voiceprints.
* Compare one person’s voice with another voice.
* Enroll a voice in an identity database.
* Identify a particular speaker.
* Verify a person’s identity from a voice.
* Provide voice-based login.
* Recognize returning users by voice.
* Attribute a sound to a named person.

General frequency or volume measurements used temporarily for game control are not intended to identify a specific individual.

The project must receive a separate technical, privacy, and legal review before any biometric, speaker-recognition, or voice-authentication feature is added.

## 10. No health, emotion, or mental-state analysis

The project is not a medical, psychological, diagnostic, therapeutic, or health-monitoring service.

It is not designed to determine or infer a person’s:

* Physical health.
* Mental health.
* Medical condition.
* Psychological condition.
* Emotional state.
* Intentions.
* Truthfulness.
* Identity.
* Consciousness.
* Thoughts.
* Telepathic ability.

Terms such as “telepathic,” “mood,” “soul,” “awareness,” or similar language appearing in the artwork, story, character names, graphics, or source code are fictional, artistic, or metaphorical. They are not scientific measurements or claims about a user.

The project does not provide medical advice and should not be used to make health, safety, employment, housing, educational, insurance, legal, or financial decisions about a person.

## 11. Children’s privacy

Telepathic-Immortal is intended as a general-audience experimental art and game project. It is not designed, marketed, or directed specifically to children under 13.

The application is not intentionally designed to collect names, ages, dates of birth, accounts, contact information, user uploads, stored voice recordings, or other personal information from children.

Children under 13 should not activate a microphone-enabled feature without the permission and supervision of a parent or legal guardian.

A supervising adult should make sure that:

* The child understands when the microphone is active.
* The microphone is stopped after use.
* No private conversation is occurring nearby.
* Everyone whose conversation could be captured has agreed.
* The browser’s microphone permission is removed when appropriate.

This section describes the project’s intended audience and design. It is not a certification or guarantee of compliance with the Children’s Online Privacy Protection Act or any other law.

The project’s privacy requirements must be reviewed again before adding child-directed marketing, accounts, uploads, advertising, analytics, persistent identifiers, cloud processing, stored recordings, or personal-information collection.

## 12. Data retention and deletion

The project’s application code is not intentionally designed to create a server-side user database or retain microphone recordings.

Because the application is not designed to transmit personal information to the project author, the project author ordinarily has no application-level user record to locate or delete.

A user may remove information stored locally by a browser, when applicable, by:

* Closing or reloading the page.
* Clearing site data.
* Clearing browser storage.
* Revoking microphone permission.
* Removing downloaded copies of the project.

A website host or other third party may maintain its own security logs or technical records according to its own retention practices.

## 13. Data sharing and sale

The project author does not intentionally sell microphone audio, microphone-derived values, or user personal information.

The project’s current application code is not designed to share microphone audio or microphone-derived values with advertisers, data brokers, analytics companies, or other outside recipients.

This does not describe or control independent processing by the user’s browser, operating system, internet provider, website host, browser extensions, or other third-party software.

## 14. Security

The project is designed to reduce privacy risk by avoiding application-level accounts, stored recordings, advertising systems, analytics systems, and server-side microphone processing.

However, no software, browser, network, website host, or computing device can be guaranteed to be completely secure.

Users should:

* Use a current browser.
* Review the browser’s permission prompt.
* Grant microphone access only when needed.
* Stop microphone access after use.
* Avoid using the microphone around private conversations.
* Review installed browser extensions.
* Use a trusted device and network.
* Download project files only from a trusted source.

Security problems involving the project may be reported using the contact information below.
